# コードレビューの実施

PR または変更差分に対してコードレビューを行う。

## レビュー観点

- **正確性**: バグ、エッジケース、エラーハンドリングの適切さ
- **セキュリティ**: SQL インジェクション、XSS、認証・認可の問題
- **パフォーマンス**: N+1 クエリ、不必要なループ、メモリリーク
- **可読性**: 変数名・関数名・コメントの適切さ
- **設計**: 単一責任の原則、不必要な複雑性

## ルール

- 指摘は具体的なファイルと行番号を示す
- 重要度を示す（Critical / Warning / Info）
- 修正案を提示する
- 問題がなければ LGTM と記載する

## 手順

1. `gh pr diff` または `git diff` で変更内容を確認する
2. 変更されたファイルを Read ツールで読み込む
3. 各変更点をレビューする
4. レビュー結果をまとめて報告する
