# AI コーディングルール

## 言語
- 日本語で回答する
- コードや技術用語は英語のまま

## コーディング方針
- シンプルで最小限の変更を優先する
- 明示的に求められていないリファクタリング・機能追加はしない
- 自明でないロジックにのみコメントを追加する
- エラーハンドリングはシステム境界（ユーザー入力・外部API）にのみ行う
- 過剰な抽象化・将来の要件に向けた設計はしない

## ツール使用
- 専用ツールが利用可能な場合は Bash より専用ツールを優先する
  - ファイル検索: Glob（find/ls の代わり）
  - 内容検索: Grep（grep/rg の代わり）
  - ファイル読み込み: Read（cat/head/tail の代わり）
  - ファイル編集: Edit（sed/awk の代わり）
- 独立した操作は並列で実行する
- サブエージェントはコンテキストが独立している場合のみ使用する

## Git
- コミットは明示的に求められた場合のみ作成する
- `git add -A` や `git add .` ではなくファイル名を指定してステージングする
- プッシュはユーザーが確認してから行う
